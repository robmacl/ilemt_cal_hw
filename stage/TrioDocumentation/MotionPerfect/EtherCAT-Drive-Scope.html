EtherCAT Drive Scope

The Scope page implements a graphical view of motion on the drive.  Allowing data capture on the drive which can sample data at up to 4kHz, one sample every 125 microseconds.  This  is then uploaded and can be displayed and maniplulated in the Scope screen.

The Scope screen is divided into 3 sections:

Configuration

Display

Move

Configuration

The configuration panel is used to set up the data capture on the drive.  There are multiple data capture buffers on the drive which can be used to capture selected items at a specified rate.  This is used to build up a time-based graph of the drive operation.

Capture

The non channel-based capture setting are:

  • Resolution: rate at which each capture point is sampled.  This can be specified in multiples of 125us and will be rounded down once the capture is started
  • Trigger Mode: selects the trigger method used to start the capture. 
    • NoTrigger (default) – manually start the scope
    • Rising – start capture when trigger channel value changes from less than trigger threshold to greater than trigger threshold
    • Falling – start capture when trigger channel value changes from greater than trigger threshold to less than trigger threshold
    • Change – start capture when trigger channel value changes from the value of trigger threshold
    • Error – start capture when drive is in error
    • GreaterThanThreshold – start capture when trigger channel value is greater than the trigger threshold
    • LessThanThreshold – start capture when trigger channel value is less than the trigger threshold
  • Trigger Threshold: the value used in the scope triggering
  • Trigger Channel: scope channel used as the trigger source

Channel

There are up to 8 capture channels on the drive, depending on the data being capture it may be that not all 8 can be used together.

  • Visible: show or hide the data line
  • Target: value to be captured

Target

Description

Position Command

Set point for the position loop, typically object 0x607A:0 (Target position)

Position Command Speed

Speed derived from Position Command

Position Feedback

Position from the motor encoder, typically object 0x6064:0 (Actual position)

Position Error

Difference between Position Command and Position Feedback

Speed Command

Set point for the speed loop

Speed Feedback

Speed from the motor encoder

Current Command (Iqr)

Set point for the torque generating current loop

Current Feedback (Iq)

Measured current

  • Color: color of the line on the graph
  • Scaling: show scaling options pop-up to modify how the displayed data line is scaled. 
    • Auto – automatic scaling
    • Same as – use same scaling as another channel.  This is often useful is there are multiple lines showing values with common units, e.g. speed command, speed feedback.
    • Manual – set the offset and amplitude manually to adjust the position of the data line
  • Cursor: show cursor options pop-up.  The channel-based cursors allow easy measurements to be taken from the graph. 
    • Enable – show or hide the cursor
    • Cursor Mode – select either horizontal or vertical cursors
    • Cursor 1 – position of cursor 1
    • Cursor 2 – position of cursor 2
    • Auto label position – move measurement label box to default location

Display
