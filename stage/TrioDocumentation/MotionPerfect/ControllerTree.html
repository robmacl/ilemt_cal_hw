Controller Tree

The controller tree can be displayed when Motion Perfect is operating in "Tool Mode" or in "Sync Mode". It contains information about the controller connected to Motion Perfect and its contents.

The tree consists of a header section and the tree body.

Tree Header

The tree header contains basic information about the controller plus some important controls. The top of the header contains a pictorial representation of the controller, the controller model (MC464 in the case above), the product number of the controller (if applicable), the system software version number, an "Axis Status" indicator, a System Status indicator and a "Reset MC" control. The bottom of the header contains three button controls: "Motion Stop", "Drive Enable" and "Halt Programs"

Controller Information

The controller is shown as an icon to the left of the header. The controller model and system software version are displayed towards the top of the header. If the mouse cursor is moved over the icon a tooltip is displayed giving some basic information about the controller.

"Axis Status" Indicator

This control shows the axis error status of the controller. It is a passive control when there is no error, displaying the message “Axis Status: OK”. When an error occurs the control becomes coloured red and acts as a button which, when clicked, will clear the axis error(s) on the controller.

Some errors, notably hardware errors, cannot be cleared by clicking the "Axis Status" button.

"System" Status Indicator

This control shows the system error status of the controller. It is a passive control when there is no error, displaying the message “System: OK”. When an error occurs the becomes coloured red and acts as a button which, when clicked, will attempt to clear the system error(s) on the controller, although in many cases this is not possible and some other manual intervention will be required.

"Reset MC" Control

This control acts as a status indicator and a reset button. If the MC_CONFIG program on the controller has been created, changed or deleted since the controller was last reset, the control is highlighted   indicating that the controller needs to be reset before programs can be run again. Otherwise the control appears as a normal button. When in either state, clicking on the control will reset the controller.

“Flash” Status Indicator

This indicates, by turning red, when the controller is writing to its “flash” EEPROM memory.

Do not remove power from the controller whilst the “Flash” status indicator shows red as doing so may corrupt the contents for the controller’s memory.

"Motion Stop" Button

Clicking on the "Motion Stop" button stops all currently running programs and empties all the move buffers on the controller causing all motion to stop. Its action is similar to an "Emergency Stop" button but, as it is implemented in software, it is less reliable that a properly implemented hardware emergency stop.

It is important that a proper hardware emergency stop is implemented on any system. This button must not be used as a substitute.

"Drive Enable" Button

Clicking on this button toggles the state of the drive enable (watchdog output) on the controller. When drives are enabled the background of the button is coloured yellow.

"Halt Programs" Button

Clicking on this button halts all currently running programs but does not stop current or buffered moves.

Use the "Motion Stop" button if you want to stop the motion as well as the programs.

Tree Body

The body of the tree contains information in several expandable sections:

Section Name

Contents

Programs

Programs and files stored on the controller.

Temporary Files

A list of the temporary currently files on the controller.

Axes

Maximum axis information plus a list of the axes defined as visible.

Memory

Memory related information.

Modules

Interface modules connected to the controller.

This is only displayed if the controller supports expansion modules.

Configuration

Controller configuration information.

Programs

These are the programs and files stored on the controller. The following types of item can be stored on the controller:

  • TrioBASIC program
  • TrioBASIC library
  • Text file
  • MC_CONFIG program (one only)
  • EC_EXTEND configuration (one only)
  • Cam Library - cam profile(s) produced by CamGen (one only).
  • HMI project containing one or more HMI page definitions (one only and not available on all controllers).
  • HMI library containing one or more HMI page definitions to be used by HMI project (not available on all controllers).
  • IEC 61131-3 task containing one or more programs in one or more of the IEC 61131-3 defined program types (not available on all controllers).
  • IEC 61131-3 library containing one or more programs in one or more of the IEC 61131-3 defined program types (not available on all controllers).
  • Robot program
  • Robot TrioBASIC program

In this section more than one item can be selected to participate in the subsequent operations.

It is now possible to group one or more items in a project based on some criteria that user defines. Movement to into or out of group can be done via drag & drop.

A package can later be created from the items in a group and all items are encrypted during package creation process. For the items to become part of the package they have to be encryptable.

Robot program and robot TrioBASIC programs automatically become part of robot project which is dedicated group inside the main project. Robot project allows set of actions like save or load to be applied to all items inside it.

Axes

Shows axis information in the form of a list. This includes:

  • The maximum number of various types of axis is given in a list.
  • Axes which are specified as visible by the user. To specify which axes are to be shown, right click on axes and select  "Show/Hide Axes…" to display the "Show/Hide Axes" dialog and select which axes to display. The assigned axis names (if specified) are displayed along with the axis number.

Memory

This shows various memory related items as follows:

VR

The maximum number of VR variables allowed. Double clicking on this launches the VR Viewer tool.

TABLE

The size (in values) of the TABLE memory area. Double clicking on this launches the Table Viewer Tool.

Local Variables

Double clicking on this launches the variable viewer tool.

Free Program Space

The number of bytes of unused memory available for storing programs.

Modules

This give a list of the modules connected to a controller. Currently this only supports the local modules of a modular controller such as the MC464.

Configuration

Shows the current controller configuration and allows the user to change some user configurable features, such as enabled features (using feature enable codes) and communications link parameters.
