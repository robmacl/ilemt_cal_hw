Packages

Packages are available from version 4.3 of Motion Perfect.

A package is a bundled set of protected (encrypted) project items. It is usually developed and published to provide some functionality to project developers, like BASIC libraries, programs, HMI content, etc. A package does not reveal its source code and thus protects the intellectual property of the author.

The main features of a package are:

  • A package is an atomic way to provide some base/platform implementation to other developers.

•         It has a protected content to hide/protect the intellectual property of the author.

•         Items from the package are logically grouped in the project.

•         Its content is used “as-is” and cannot be changed or viewed.

•         Usually it can be safely replaced (updated) in the user project with a newer version provided (published) by the package developer. Motion Perfect provides the functionality to update packages.

•         More than one package can be used in a project. Each package may require a different project key.

•         If the package includes HMI content, then an appropriate Uniplay licence file will be needed by the Uniplay device to enable it to load and display the content from the package.
