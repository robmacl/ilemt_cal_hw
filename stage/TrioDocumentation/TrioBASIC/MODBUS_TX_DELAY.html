MODBUS_TX_DELAY

Type:

System Parameter

Syntax:

MODBUS_TX_DELAY = <value>

Description:

This parameter operates on the Modbus RTU (serial) server. It is the delay in milliseconds between the incoming Modbus request packet and the outgoing response. There is a built-in delay of approximately 3 ms. The value entered in this parameter will be added to the normal delay.

Set this delay if the HMI needs some time to switch to receive mode.  Especially useful to tune the timings on RS485 2-wire Modbus.

Parameters:

value:

Time in milliseconds between end of request and start of response.

Examples:

Example 1:

The HMI needs a 10 ms delay between sending its request packet and starting to receive the response from the Motion Coordinator. Uses port 2 in 2-wire RS485 mode.

MODBUS_TX_DELAY = 7 'There is a built-in 3 ms delay

ADDRESS = 1
