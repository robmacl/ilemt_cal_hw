TARGETJ

Type:

Robotics Type

Syntax:

DIM <var-list> AS TARGETJ

<target-variable> = x, y, z, u, v, w

<target-variable> = <target-variable>

Description:

TARGETJ is a datatype to store joint positions and is commonly used with robotic move commands, such as MOVEC , MOVEJ and MOVEL . TARGETJ variables are for local use only and can be defined just as other variables in a program. For global use, use the global target array (see GTAJ ).

It is possible to have an array of local targets by defining the dimensions it in the declaration statement:

DIM point4 AS TARGETJ(10)

It is possible read from and write into the target’s elements using the ‘dot’ notation (“.”) as shown in the table below:

x_val = tgt.x

tgt.y = 12.2

Assignment between targets, both local and global, is possible:

GTAJ(20) = GTAJ(14)

GTAJ(10).x = GTAJ(11).x

my_target_1 = GTAJ(10)

GTAJ(11) = my_target_2

TARGETJ requires the RPS runtime FEC .  

Parameters:

Name

Description

x, y, z, u, v, w: Float

Joint positions x, y, z, u, v and w (mm or degrees as required by the robot type)

The naming convention x, y, z, u, v, w was chosen to reflect Cartesian coordinates, as this was originally their sole purpose. Joint targets expanded this capability but did not rename the members to make them joint specific, so their naming may seem confusing.

Examples:

Example 1:

Define an array of 4 targets.

DIM trgts AS TARGETJ(4)

trgts(0) = 45.0, 0.0, 270.0, 0.0, 90.0, 0.0

trgts(1) = 180.0, 270.0, 180.0, 0.0, 90.0, 0.0

trgts(2) = -90.0, 300.0, 180.0, -90.0, 90.0, 0.0

trgts(3) = 45.0, 270.0, 180.0, -90.0, 90.0, 0.0
