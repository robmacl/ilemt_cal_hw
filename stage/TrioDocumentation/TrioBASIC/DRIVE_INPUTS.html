DRIVE_INPUTS

Type:

Axis Parameter

Syntax:

Value = DRIVE_INPUTS AXIS(n)

Description:

Returns a numerical value that represents the bit pattern of the digital inputs on a remote axis. EtherCAT axes commonly have either a 16-bit word or 32-bit long-word in the cyclic data that is mapped to the Drive’s digital inputs. The data may also be mapped to the Motion Coordinator I/O Map.

Examples:

Example 1:

In the Motion Perfect command line terminal, find the bit pattern of the drive inputs on axis 2.

>>BASE(2)
>>?HEX(drive_inputs)
90000

Bits 16 and 19 are 1. The slave has inputs offset by 16 bits, so this is inputs 0 and 3 on the drive.

Example 2:

A program must wait for input 3 to be ON. Input 3 is mapped to bit 19 in DRIVE_INPUTS.

BASE(2)

WAIT UNTIL (DRIVE_INPUTS AND $80000) <> 0

Note that the inputs are also in the I/O map so WAIT UNTIL IN(x) = ON will do the same job.  Where x is the IN number mapped to the bit in the drive cyclic data.
