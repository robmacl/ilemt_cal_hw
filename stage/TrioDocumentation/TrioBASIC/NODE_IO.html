NODE_IO

Type:

System Array (MC_CONFIG)

Syntax:

NODE_IO(slot, node_address) = offset_value

Description:

This 2D array can be used to set the start address of any EtherCAT node I/O channels. This can be used to define a user specific I/O map to fix I/O points from different sources in place.

The array is 2-dimensional, the first dimension is the master slot identifier, the second dimension is the position of the node within that master network.

The default I/O allocation order is: Controller I/O, Module I/O, CAN I/O, EtherCAT I/O (both drive I/O and independent EtherCAT I/O)

When using multiple I/O types, it is best to configure the I/O order with the I/O base modifier parameters.  See below.

Parameters:

slot:

The slot number of the EtherCAT port.  (0 on MC4N, MC6N and Flex-6 Nano)

node_address:

Address determined by the position of the node on the EtherCAT network.  The first node is address 0.

offset_value

0

EtherCAT I/O allocated automatically (default)

>= 8

EtherCAT I/O for the defined Node starts at this I/O point address

To allocate I/O offsets based on the EtherCAT configured address, use NW_NODE_IO

Example:

A system with MC464, an EtherCAT module (slot 0) and a 128 inputs EtherCAT I/O module at node 5 will normally have the following I/O assignment:

MODULEIO_BASE = 0: NODE_IO(0, 5) = 0: CANIO_BASE = 0
