MOTION_ERROR

Type:

System Parameter (read only)

Syntax:

value = MOTION_ERROR

value = MOTION_ERROR(<bank>)

Description:

The MOTION_ERROR system parameter is a 64-bit integer that provides an indication that one or more axes are in error, with a bit being set for each axis in error. Since this parameter is a bit mask, it is usual to use the bit operator to interrogate each bit.

In order to handle up to 1024 axes, MOTION_ERROR is split into 16 banks of 64-bit integers with each bank containing the motion error state of 64 axes. To access a bank, MOTION_ERROR can be indexed using an index from 0 to 15. As with the unindexed form, the bit operator may be used in conjunction with indexing.

Parameters:

bank

The bank of axes to access, range 0 to 15 (default 0).

Value:

A sum of the bit values representing each axis that is in error.

Bit

Axis

0

0 in bank 0; 64 in bank 1; 128 in bank 3; ...

1

1 in bank 0; 65 in bank 1; 129 in bank 3; ...

2

2 in bank 0; 66 in bank 1; 130 in bank 3; ...

3

3 in bank 0; 67 in bank 1; 131 in bank 3; ...

…

 

63

63 in bank 0; 127 in bank 1; 191 in bank 3; ...

Example:

Example 1:

Axes 0, 1 and 3 have an error and the error on axis 3 occurred first.

>>?MOTION_ERROR
11
>>?ERROR_AXIS
3

Example 2:

When motion error occurs, this program routine checks the axis status for each axis in error and prints a message to terminal 5.

IF MOTION_ERROR <> 0 THEN

  FOR a = 0 to 23

    IF MOTION_ERROR.a = 1 THEN
      IF AXISSTATUS AXIS(a).2 = 1 THEN

        PRINT #5, "Communication error on axis "; a[0]

      IF AXISSTATUS AXIS(a).3 = 1 THEN

        PRINT #5, "Remote drive error on axis "; a[0]

      IF AXISSTATUS AXIS(a).8 = 1 THEN

        PRINT #5, "FE exceeded FE_LIMIT on axis "; a[0]
      ENDIF

    ENDIF

  NEXT a
ENDIF

Example 3:

Display the motion error state of axis 70 on the command like

PRINT MOTION_ERROR(1).6
