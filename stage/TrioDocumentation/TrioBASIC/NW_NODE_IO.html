NW_NODE_IO

Type:

System Array (MC_CONFIG)

Syntax:

NW_NODE_IO(slot, alias_address) = offset_value

Description:

This 2D array can be used to set the start address of any EtherCAT node I/O channels. This can be used to define a user specific I/O map to fix I/O points from different sources in place.

The default I/O allocation order is; Controller I/O, Module I/O, CAN I/O, EtherCAT I/O (both drive I/O and independent EtherCAT I/O)

When using multiple I/O types, it is best to configure the I/O order with the I/O base modifier parameters. See NODE_IO.

Parameters:

slot:

The slot number of the EtherCAT port.  (0 on MC4N, MC6N and Flex-6 Nano)

alias_address:

The configured alias address of the slave device.

offset_value:

0

EtherCAT I/O allocated automatically (default)

>= 8

EtherCAT I/O for the defined Node starts at this I/O point address

Example:

A system with MC664, using the built-in EtherCAT port (slot 0) and  5 EtherCAT servo drives with digital I/O. Configured Station Alias addresses are 1 to 5. It is required that if one drive is removed, the digital I/O do not change. Each servo drive has 16 inputs and 16 outputs.

NW_NODE_IO(0, 1) = 32

NW_NODE_IO(0, 2) = 48

NW_NODE_IO(0, 3) = 64

NW_NODE_IO(0, 4) = 80

NW_NODE_IO(0, 5) = 96
