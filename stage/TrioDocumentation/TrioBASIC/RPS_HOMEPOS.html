HOMEPOS

Type:

Robotics Command

Syntax:

HOMEPOS(position_1 [,position_2 [,… [, position_n]])

Description:

Specifies the home position of an axis. Typically, this is used with robot axes and the values used are supplied by the robot manufacturer when the robot was calibrated.

Home position is the position of a robot axis at its zero position, which aligns to the calibration mark of the relevant robot’s joint. For axes with a multi-turn encoder, this should be the position within one turn of the axis.

HOMEPOS calculates the difference between the actual position and the referenced position parameter to work out the position of the joint in base coordinate system.

HOMEPOS works like OFFPOS in the way that it applies the offset, but unlike OFFPOS is not incremental, so the same home position can be applied multiple times with the same position and only the first setting will have any affect.

The value position is placed in DPOS , while MPOS is adjusted to maintain the FE value.  This function is completed after the next servo-cycle. 

Typically, HOMEPOS is used for calibration. Therefore, use HOMEPOS only when joints are aligned with calibration marks of the robot.

Incorrect positioning of robot joints during execution of HOMEPOS command can result in incorrect movement of the robot.

Parameters:

position_1:

Position to set on current base axis in user units. Typically, this is the encoder value at zero position of the axis.

position_2:

Position to set on next axis in BASE array in user units. Typically, this is the encoder value at zero position of the axis.

position_n:

Position to set on last axis in BASE array in user units. Typically, this is the encoder value at zero position of the axis.

Examples:

Example 1:

Axis 1 has a 23-bit-encoder connected to the motor. It has a gearbox ratio of 80.0057. Calibrate the position of axis 1 which has encoder value of 2053664 at position 0.

BASE ( 1 )
AXIS_UNITS = ( 2 ^ 23 ) * ( 800057 / 10000 ) / 360  'Units in degrees
HOMEPOS ( 2053664 )  'Zero Position encoder value

Example 2:

A SCARA robot is configured to have axes 1, 3, 5 and 7 as its joints 1, 2, 3 and 4 respectively. Calibration values for these joints are 2053664, 4543662, 860240 and 1635534 respectively.

BASE (1, 3, 5, 7)
HOMEPOS (2053664, 4543662, 860240, 1635534)

Example 3:

Display the HOMEPOS values previously applied to axes 0, 1 and 2.

PRINT HOMEPOS AXIS(0)

PRINT HOMEPOS AXIS(1)

PRINT HOMEPOS AXIS(2)
