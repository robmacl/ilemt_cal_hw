DEBUG_LOG

Type:

System Function

Syntax:

DEBUG_LOG([start[, finish]])

Description:

Records up to 4096 of the most recent controller optional events.

Each event is recorded with a timestamp of the current RTC value and the time since power-up of the controller.

Each event is also tagged with a checksum for verification.

When executing the command, either in the terminal or a Trio BASIC program, it will print the recorded event messages to the terminal.

Note : Before displaying a log entry the checksum for the log is validated, if it is invalid then the message '<INVALID LOG ENTRY>' will be displayed instead.

List of events recorded:

A list of the events and its corresponding recorded messages is configured by the DEBUG_LOG_MODE parameter.

Parameters:

start:

Displays all debug log entries starting from line number 'start'.

-1

Special case, erases log from flash memory (second parameter is not needed).

finish:

Displays all debug log entries starting from line number 'start' until line number 'finish'.

Example:

Example log file output:

DEBUG_LOG(1,7)

DEBUG_LOG_MODE , FLASH_LOG
