=  Equal

Type:

Comparison Operator

Syntax:

<expression-1> = <expression-2>

Description:

Returns TRUE if <expression-1> is equal to <expression-2>, otherwise returns FALSE .

When comparing floating-point values, remember that they tend not to be exactly equal apart from in special circumstances, so you will usually have to compare them using an alternative method, for example an absolute or relative difference combined with a suitably small tolerance.

The comparison operators are typically used in conditional statements like IF , REPEAT and WHILE , though can be used anywhere an expression is valid.

Parameters:

Expression1:

Any valid TrioBASIC expression

Expression2:

Any valid TrioBASIC expression

The following TrioBASIC expressions are valid:

Expression 1

Expression 2

Description

NUMERIC

NUMERIC

FLOAT, INTEGER, IP

STRING

STRING

Case sensitive lexical comparison

It is not permitted to compare arrays.

Examples:

Example 1:

If input 7 is ON then program execution will continue at line starting “label:”

IF IN(7) = ON THEN GOTO label

Example 2:

Call subroutine ‘threshold_1’ when analogue input 0 reads 2.1 V.

This example avoids the direct comparison of a floating-point value (IF AIN(0) = 2.1 THEN…), as this is unlikely to ever be true, instead using the absolute value of an absolute difference combined with a comparison tolerance.

IF ABS(AIN(0) – 2.1) < 0.5e-6 THEN GOSUB threshold_1
