GTAJ

Type:

Robotics Parameter

Syntax:

GTAJ(index) = x, y, z, u, v, w, name

Description:

GTAJ is the Global Targets Array which contains an array of globally available TARGETJ objects for use in Robotics applications.

GTAJ variables, like GTA and GTAE, are global to all programs and their values are retained. For local targets, see TARGETJ.

A GTAJ variable is an aggregate of multiple values, each value being stored in a named member as shown below.

Keyword

Description

x, y, z, u, v, w

64-bit floating point number

id

32-character string

active

Boolean

Global targets may be activated or deactivated using the Boolean member “active”. Assigning ‘FALSE’ will erase the target (all values to 0 and empty name) and deactivate it (not visible in any program). When a target has been deactivated, it is like it had never been used, so deactivating a target can be done with any of GTA, GTAJ or GTAE and the result will be the same.

GTAJ ( 10 ).active = FALSE

If it is required to reset all elements of the global target array, then the following command can be used.

GTAJ ( 0, GTASIZE - 1 ).active = FALSE

A range of GTAJs can be requested or assigned by specifying the start and end index separated by comma.

GTAJ ( 555 , 559 ) = 90 , - 90 , 0 , 0 , 0 , 0
PRINT GTA ( 555 , 559 )

Global target members can be read from and written to using the ‘dot’ notation (“.”).

my_variable = GTAJ ( 10 ).x

GTAJ(10).x = (GTAJ(23).x + GTAJ(24).x) / 2

If an inactive global target array element is assigned to using GTAJ, then that element becomes a GTAJ, i.e. has a point type of 1, and must be manipulated with GTAJ otherwise a run-time error will be generated. It is also permissible to use GTAE for any point type, but it is a good general rule to know what the point type is for an element and to always use the appropriate access keyword.

Global target array elements may be assigned to each other and to and from local target variables.

GTAJ ( 10 ) = GTAJ ( 11 )

my_target = GTAJ ( 10 )

GTAJ(12) = my_target

Once Global target array elements may be assigned to each other and to and from local target variables.

GTAJ requires the RPS runtime FEC .

GTA, GTAJ, and GTAE are sharing the same allocated location of the memory. This means they cannot share indices. Maximum number of targets is controller dependent. 

For example: 
GTA(1) = X1, Y1, Z1, U1, V1, W1, “pt1” 

GTAJ(2) = t1, t2, t3, t4, t5, t6, “pt2” 

In this example, GTAJ cannot have index 1 and GTA cannot have index 2. 

Parameters:

Name

Description

x, y, z, u, v, w: Float

Positions in Joint space (degrees).

(optional) “name”: String

Unique name for the target. Duplicated names generate a compile error.

Name is optional but if specified must be unique (not used by any other GTA elements of any type).

Name is truncated if more than 32 characters long.

A GTAJ cannot be declared using a name; instead, it must use an id number. If so, the run time error “ID not defined” will be thrown.

Examples:

Example 1:

Define a global target named “Joint_point1”.

GTAJ (10) = 100 , 0 , 50 , 180 , 0 , 180 , "Joint_point1"
