TARGET

Type:

Robotics Type

Syntax:

DIM <var-list> AS TARGET

<target-variable> = x, y, z, u, v, w

<target-variable> = <target-variable>

Description:

TARGET is a datatype to store position and orientation and is commonly used with robotic move commands, such as MOVEC , MOVEJ and MOVEL . TARGET variables are for local use only and can be defined just as other variables in a program. For global use, use the global target array (see GTA ).

It is possible to have an array of local targets by defining the dimensions it in the declaration statement:

DIM point4 AS TARGET ( 10 )

It is possible read from and write into the target’s elements using the ‘dot’ notation (“.”) as shown in the table below:

x_val = tgt.x

tgt.y = 12.2

Assignment between targets, both local and global, is possible:

GTA(20) = GTA(14)

GTA(10).x = GTA(11).x

my_target_1 = GTA(10)

GTA(11) = my_target_2

TARGET requires the RPS runtime FEC .

Parameters:

Name

Description

x, y, z, u, v , w: Float

Cartesian positions x, y and z (mm), and orientations u, v and w (degrees)

Examples:

Example 1:

Define an array of 4 targets.

DIM trgts AS TARGET(4)

trgts(0) = 433.0, 0.0, 370.0, 0.0, 90.0, 0.0

trgts(1) = 433.0, 300.0, 370.0, 0.0, 90.0, 0.0

trgts(2) = -100.0, 300.0, 370.0, -90.0, 90.0, 0.0

trgts(3) = 433.0, 300.0, 370.0, -90.0, 90.0, 0.0
