COORDINATOR_DATA

Type:

System Parameter

Syntax:

COORDINATOR_DATA(function_number[, param_values])

Description:

Allows special firmware features to be enabled or changed. Check the function you intend to use is available by first contacting your Trio Distributor.

The random use of undocumented COORDINATOR_DATA functions may cause the Motion Coordinator to become unstable. It will then need to be returned to the manufacturer for reset.

Value:

Function number.  Each value represents a different operation.

Function

Description

Value

16

Disable/Enable BASE check on axes order. When enabled, the axes in BASE must be in ascending order, e.g. BASE(0, 1, 4, 5, 6) 

0 = Axes order check enabled. (Default) 
1 = Axes order check disabled.

0 / 1

30

Returns the P number of the CANIO module at address n

 

31

Returns the firmware Boot Code version number

 

53

Over-rule the system that prevents software from loading a BASIC or Text file into the Motion Coordinator when Motion Perfect is connected in Sync mode.

0 / 1

Examples

Example 1:

Allow the Motion Coordinator program to allocate an axis group in either descending or mixed axis order.

COORDINATOR_DATA(16, 1)

 

The programs may now use any axis order for BASE.

BASE(4, 3, 0, 1)

Note: When using Kinematic FRAMEs, the axes must be in ascending order.

Example 2:

Get the P number of the digital CAN I/O module at address 0 and the analogue CAN I/O module at address 16.

>>?COORDINATOR_DATA(30, 0)
318
>>?COORDINATOR_DATA(30, 16)
326

Example 3:

While debugging a program that loads files into the synchronised part of the memory, we want to load programs from an external source at the same time as Motion Perfect is connected in Sync Mode. This line is added to the BASIC program which runs on power up.

COORDINATOR_DATA(53, 1) 'Allow files to be loaded while in Sync connection mode to MP
