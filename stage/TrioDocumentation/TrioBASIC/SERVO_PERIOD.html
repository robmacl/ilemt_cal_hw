SERVO_PERIOD

Type:

System Parameter (MC_CONFIG / FLASH)

Description:

This parameter allows the controller servo period to be read or specified. This is the cycle time in which the target position updated and if applicable any positions are read, and closed loop calculations performed.

SERVO_PERIOD is a flash parameter and so should be set using the MC_CONFIG file.

When the servo period is reduced the maximum number of axes (including virtual) is reduced as per the following table.

The number of supported axes for each SERVO_PERIOD value is tested with optimal conditions and light EtherCAT loads, the number supported in a real system may vary.

Value:

SERVO_PERIOD is specified in microseconds. Only the values above may be used, and the Motion Coordinator must be reset before the new servo period will be applied.

The permissible number of axes decreases as the SERVO_PERIOD is reduced, and this is because the servo period defines the time available within which processing of all the axes in use must be complete before the next servo cycle starts. Normally, the maximum number of axes a controller supports can be used when SERVO_PERIOD is set to 1 ms or higher.

Examples:

Example 1:

'Check controller servo period on startup

IF SERVO_PERIOD <> 250 THEN

  SERVO_PERIOD = 250

  EX

ENDIF

Example 2:

' MC_CONFIG script file

SERVO_PERIOD = 500 'This is the value set on power up
