INCLUDE

Type:

System Command.

Syntax:

INCLUDE "filename"

Description:

Allows the inclusion of TrioBASIC commands and constructs (except ON BASICERROR). This is typically used to share data initialisation between programs, but you can imagine an INCLUDE as a copy and paste from the include into the program at the point of the INCLUDE statement, but with the benefit that any changes made to the included file are shared between all programs that include it.

Whenever an included program is modified, all programs that depend on it are re-compiled as well, avoiding inconsistencies.

Note that prior to firmware version 2.0269:

•         Nested includes were not allowed (now they are allowed to be up to 5 deep)

•         Includes had to be the first statement (barring comments) in a program (now they may be placed anywhere)

•         Only variable definitions and conditional logic were allowed, and it could not be used as a general subroutine with any commands within it (it may now contain any statements)

Circular dependency is not permitted. So, A cannot include itself; if A includes B, B cannot include A; if A includes B and B Includes C, C cannot include either B or A, etc.

Parameters:

filename:

The name of the program to be included

Example:

Initialise all local variables with an include program.

Program “MOTION_1”:

'Include data definitions

INCLUDE "DATA_DEFS"

 

'Motion commands using included data

FORWARD AXIS(drive_axis)

CONNECT(1, drive_axis) AXIS(linked_axis)

Program “DATA_DEFS”:

drive_axis = 4

linked_axis = 1
