NW_NODE_PROFILE

Type:

System Array (MC_CONFIG)

Syntax:

NW_NODE_PROFILE(slot, alias_address) = value

Description:

This is an array system parameter which can be used in MC_CONFIG.  The parameter is used to define which profile number should be mapped to a particular EtherCAT slave. The EtherCAT protocol startup will fail if the selected profile is not available or is out of range.

The NW_NODE_PROFILE profile ID values are initialised to zero, because any profile ID value will only be used if a slave device has been setup with the same non-zero configured station alias.  Zero valued station alias addresses are not used by the EtherCAT protocol.

If set to non-zero, the use of NODE_PROFILE for the same slave device will be used in preference to a NW_NODE_PROFILE.

Parameters:

slot:

The slot number of the EtherCAT port. (0 on MC4N, MC6N and Flex-6 Nano)

alias_address:

Configured address of the node.

value:

0

Use the default node profile/configuration (default)

>= 1

Use the specified EtherCAT profile/configuration

Example:

Set the profile of the P375 at alias address 123 to 2.  Equivalent to DRIVE_PROFILE = 2.

In MC_CONFIG:       

NW_NODE_PROFILE(0, 12) = 2

Start the EtherCAT network.  Terminal #0 output shows:

EtherCAT State: Init (0)

EtherCAT State: Init (0)

EtherCAT State: Pre-Operational (0)

EtherCAT State: Safe-Operational (0)

EtherCAT State: Operational (0)

EtherCAT Configuration (0):

 

    P600: 0 : 0 : 2000

    P366 Coupler: 1 : 0 : 2001

    P375 3 Axes Stepper Output: 2 : 12 : 12 (11:2:0,12:2:0,13:2:0)

Check the profile number from the terminal.

    ? nw_node_profile(0,12)

    2

 

The P375 display values are:

autoincrement position : alias address : configured address (axis number : node profile ID : control mode)
