.  Field operator

Type:

Program Variable

Syntax:

<variable>.field_name

Description:

Accesses the specified field of the given variable, allowing its contents to be read or written.

A field, also known as a structure member or property, allows access to data contained within a variable by using the field’s name, which follows the same naming rules as identifiers used elsewhere to name things like variables and functions.

Fields have limited use in Trio BASIC, as there is no facility to declare arbitrary user-defined structures. The following shows where they may be used:

  • Variables of type TARGET, TARGETE and TARGETJ (see specific pages for details)
  • Variables have the following fields:
    • “dims” – returns the number of dimensions the variable has (0 if not an array)
    • “dimsize” – returns the number of elements in the dimension specified within round brackets (numbered from 0, but no parameters are allowed for non-array variables)

There should be no spaces before or after the field operator.

Parameters:

variable:

The name of a variable or function parameter

field_name:

The name of the field to be accessed

Examples:

Example 1:

Dimension a variable of type TARGETE and set the “x”, “y” and “z” fields to the measured position of axes 0, 1 and 2 respectively.

DIM t AS TARGETE

 

t.x = MPOS AXIS(0)

t.y = MPOS AXIS(1)

t.z = MPOS AXIS(2)

Example2:

Display the number of dimensions and the size of each dimension for an array function parameter.

FUNCTION test(arr AS FLOAT())

  DIM i AS INTEGER

 

  PRINT "Number of dimensions: "; arr.dims

  FOR i = 1 to arr.dims

    PRINT "Dimension "; i; ": "; arr.dimsize(i - 1)

  NEXT i

ENDFUNC

Example3:

Assign to GTA element 10 the average X coordinate from GTA elements 20 and 42.

GTA(10).x = (GTA(20).x + GTA(42).x) / 2
