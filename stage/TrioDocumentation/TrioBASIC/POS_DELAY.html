POS_DELAY

Type:

Axis Parameter

Syntax:

POS_DELAY = delay factor

Description:

To compensate for the delay in sending data across the digital drive network a delay factor can be specified that adds a position correction to the HW_PSWITCH trigger position, proportional to the current measured speed.

As an Axis Parameter, the syntax can take either BASE or the AXIS(n) modifier.

Requires special hardware functions in the Motion Coordinator FPGA.

Parameters:

delay factor

Delay time to be applied to the HW_PSWITCH position value

Examples:

Example 1:

For Panasonic A5N RTEX drives and the Trio MC4N-RTEX running with SERVO_PERIOD=1000:

POS_DELAY AXIS(2)=2.5

POS_DELAY AXIS(3)=2.5
