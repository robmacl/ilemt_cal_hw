PRINT

Type:

File Function

Alternative format:

?

Syntax:

PRINT [#channel, ]expression-1[, expression-2[, ...]]

Description:

The PRINT command allows the TrioBASIC program to output a series of characters to a channel. A channel may be a serial port or some other type of connection to the Motion Coordinator.

Multiple expressions may be printed in a single PRINT statement by separating each expression by a comma or semi-colon. Each expression may be modified using functions like HEX and CHR , and the output format may be specified using a format specifier enclosed within square brackets (see below).

In the absence of a format specifier, a value is printed to 4 decimal places.

Any value larger than 1e19 and smaller than 1e-18 will be printed using scientific notation.

Parameters:

#channel,

See # for the full channel list (default 0 or by OUTDEVICE if omitted)

Expression-n:

Any valid TrioBASIC expression, either numeric or string, scalar or array, or any of the TARGET types

The following elements may be used with an expression:

;

Punctuates expressions without any separation at all, and if used after the final expression it suppresses the output of the carriage return and line feed

,

Punctuates expressions, separating each expression with a tab character

[format-specifier]

Specifies the format that the expression should be output with, which may be either one or two numeric values, or a C format string. Format specifiers are not allowed with IP, TARGET, TARGETJ or TARGETE expressions.

Values

[d]

Number of decimal places to display

[w, d]

The total width to output (negative for left justification) and the number of decimal places

C format

“%f”

Floating-point output (64-bit)

“%F”

Floating-point output in upper case (64-bit)

“%e”

Floating-point output in scientific notation (64-bit)

“%E”

Floating-point output in scientific notation in upper case (64-bit)

“%L”

Integer output in decimal (64-bit)

“%W”

Unsigned integer output in decimal (64-bit)

“%X”

Unsigned integer output in hexadecimal (64-bit)

“%R”

String output (size + character array) – Raw byte output

“%S”

String output (size + character array) – Unicode output

“%C”

Character output

If the number is too big to fit with the specified format, the field will be filled with question marks to signify that there was insufficient space to display the number.

If the number of characters of output is less than the width, then the output is padded with spaces, either ahead of the output if right justified (positive width), or after the output if left justified (negative width).

It is preferable to use the semi-colon separator to display two strings as one than to concatenate them (i.e., “PRINT str1; str2” produces the same output as “PRINT str1 + str2”), as this will not incur the overhead of having to perform a concatenation.

C Format Strings

These are quite powerful in that they offer more formatting possibilities than when using one or two values, but at the expense of some complication.

The general format is that of a percent symbol followed by a character that represents the type of data to be output, and these are described in the table above. This provides no formatting at all, so to tailor the output produced, these two symbols may have flags, a width, and a precision placed between them, which are described below.

The flags may be in any order and the following are recognised:

“+” – Force output of the sign

“-“ – Left justify the output

“0” – Output leading zeroes

The width is a decimal value, and the precision is a decimal point followed by a decimal value. A precision may be specified without a width being specified.

String output can be controlled using two formats, “%R” and “%S”, and these are used instead of “%s” (lower case) because TrioBASIC strings are stored as a size and separate character array, whereas “%s” is of a pointer to the base of a character array terminated with a null. The default string format when using PRINT is “%R”, as this emits the characters in raw form, i.e., without any decoding, which is useful for communication protocols, but it isn’t good if you want to correctly pad output when the string is Unicode, as characters may use multiple bytes. For Unicode strings, it is better to use “%S”, as this will decode the string’s characters and apply the correct padding and precision specified in the format string. Both “%R” and “%S” parse the whole string and do not stop at the first null character.

Character output uses “%C” instead of “%c” because TrioBASIC integers are 64 bits in length and “%c” expects 8-bit values that require decoding differently. Character output is useful in avoiding having to convert the character’s code into a string before outputting it.

In addition to the formatting aspect, characters may appear ahead of the leading percent symbol and after the trailing ‘type’ symbol, and these will simply be echoed. This feature is used in the example program to show the extent of the output produced by bracketing it within colon characters.

Motion Perfect terminal

When PRINT is used to output to a Motion Perfect terminal (#0, #5, #6 or #7) cursor control ESC sequences can be used. The following VT100 type control sequences are supported from MPv4 version 4.4.1 onwards.

Cursor Control

Function

Sequence

Description

Cursor Home

<ESC>[{ROW};{COLUMN}H

Sets the cursor position where subsequent text will begin. If no row/column parameters are provided (i.e., <ESC>[H), the cursor will move to the home position, at the upper left of the screen.

Force Cursor Position

<ESC>[{ROW};{COLUMN}f

Identical to Cursor Home.

Cursor Up

<ESC>[{COUNT}A

Moves the cursor up by COUNT rows; the default count is 1.

Cursor Down

<ESC>[{COUNT}B

Moves the cursor down by COUNT rows; the default count is 1.

Cursor Forward

<ESC>[{COUNT}C

Moves the cursor forward by COUNT columns; the default count is 1.

Cursor Backward

<ESC>[{COUNT}D

Moves the cursor backward by COUNT columns; the default count is 1.

 

Erasing Text

Function

Sequence

Description

Erase End of Line

<ESC>[K

Erases from the current cursor position to the end of the current line.

Erase Start of Line

<ESC>[1K

Erases from the current cursor position to the start of the current line.

Erase Line

<ESC>[2K

Erases the entire current line.

Erase Down

<ESC>[J

Erases the terminal window from the current line down to the bottom of the window.

Erase Up

<ESC>[1J

Erases the terminal window from the current line up to the top of the window.

Erase Screen

<ESC>[2J

Erases the terminal window with the background colour and moves the cursor to home.

 

Examples:

Example 1:

Print a string using quotation marks combined with string variable.

DIM string1, string2 AS STRING(20)

string1 = "OPEN"

string2 = "CLOSED"

IF IN(4) = ON THEN

  PRINT "Guard status "; string2

ELSE

  PRINT "Guard status "; string1

ENDIF

Example 2:

Print a number and a value from a VR , separated by a comma to put the VR value in the next tab space.

>>PRINT 123.45, VR(1)

123.4500     1.5000

>> 

Example 3:

Print a VR with 4 characters and 1 decimal place, then in the next tab a local variable with 2 decimal places.

VR(1) = 6

variable = 410.5

PRINT VR(1)[4, 1], variable[2]

print output will be:

6.0     410.50

Example 4:

Print a string directly followed by a numerical value. Note how in this example the semi-colon separator is used. This does not tab into the next column, allowing the programmer more freedom in where the print items are put.

>>PRINT "POSITION = "; MPOS

POSITION = 123.0000

>> 

Example 5:

Print a carriage return and no line feed at the end of a message. The semi-colon on the end of the print line suppresses the carriage return normally sent at the end of a print line. ASCII (13) generates CR without a line feed. The string is to output from serial port channel 1.

PRINT #1, "ITEM "; total; " OF "; limit; CHR(13);

Example 6:

Print AXISSTATUS for axis 6 in the hexadecimal format on the command line.  (bits 1 and 8 are set)

>>?HEX(AXISSTATUS AXIS(6))

102

>> 

Example 7:

Print to terminal #5 using an ANSI control sequence to position the cursor before sending the text string and values.

'Need to print double escape code (ASCII 27) to output a single escape

PRINT #5, CHR(27); CHR(27);

 

'Move cursor to row 14, column 10

PRINT #5, "[14;10H";

 

'Display the temperature`

PRINT #5, "Temperature: "; AIN(1);

Example 9:

The following program shows how the “%R” and “%S” C format strings behave with Unicode strings and illustrates that while using PRINT without a format specifier (equivalent to “%R”) can output Unicode strings correctly, it cannot format them accurately.

DIM s1 AS STRING ( 100 )
DIM s2 AS STRING ( 100 )
DIM s3 AS STRING ( 100 )

s1 = "The rain in Spain falls mainly on the plain"
s2 = "Der Regen fällt in Spanien hauptsächlich auf die Ebene"
s3 = "Η βροχή στην Ισπανία πέφτει κυρίως στην πεδιάδα"

PRINT
PRINT "=== No format string (%R) ===================================="
PRINT "'"; s1; "'"
PRINT "'"; s2; "'"
PRINT "'"; s3; "'"
PRINT

PRINT
PRINT "=== %60R ====================================================="
PRINT "'"; s1["%60R"]; "'"
PRINT "'"; s2["%60R"]; "'"
PRINT "'"; s3["%60R"]; "'"

PRINT
PRINT "=== %60.30R =================================================="
PRINT "'"; s1["%60.30R"]; "'"
PRINT "'"; s2["%60.30R"]; "'"
PRINT "'"; s3["%60.30R"]; "'"

PRINT
PRINT "=== %-60.30R ================================================="
PRINT "'"; s1["%-60.30R"]; "'"
PRINT "'"; s2["%-60.30R"]; "'"
PRINT "'"; s3["%-60.30R"]; "'"

PRINT
PRINT "=== %60S ====================================================="
PRINT "'"; s1["%60S"]; "'"
PRINT "'"; s2["%60S"]; "'"
PRINT "'"; s3["%60S"]; "'"

PRINT
PRINT "=== %60.30S =================================================="
PRINT "'"; s1["%60.30S"]; "'"
PRINT "'"; s2["%60.30S"]; "'"
PRINT "'"; s3["%60.30S"]; "'"

PRINT
PRINT "=== %-60.30S ================================================="
PRINT "'"; s1["%-60.30S"]; "'"
PRINT "'"; s2["%-60.30S"]; "'"
PRINT "'"; s3["%-60.30S"]; "'"
