POSI_SEQ_MODE

Type:

Axis Parameter

Description:

The command POSI_SEQ_MODE controls the delay filter to apply the demand position (DPOS).

The delay filter allows the final axis output position (DPOS) to be fed through a delay buffer.  The delay is defined as a number of servo cycles, specified using the command POSI_SEQ_DELAY.

The filter is controlled using POSI_SEQ_MODE.  There are 3 modes of operation:

0: No delay (filter disabled)

1: Delay only.  The latest value is pushed to delay buffer and popped later when the number of specified delay cycles has elapsed.

2: 2-point filter delay.  The average of latest value pushed to delay buffer and current value popped from delay buffer.

Internally, the delay buffer must be fully populated before delayed values become active and overwrite demand position, until this happens the demand position will not be delayed.

Value:

Default value is 0.
