Kinematic Groups

KINEMATIC_GROUP command activates RPS and links the required data to the specified robot. This command must be executed to have RPS set.

M Although 8 kinematics can be initialised on a single controller, it may not be possible to process all 8 within a given SERVO_PERIOD. The total number that can be run depends on many factors such as, type of kinematic is selected, drive connection method, if OBJECT_FRAME, ROBOT_FRAME, TOOL_OFFSET or COLLISION_OBJECT is enabled and additional factory communications.

 

The number of axes in the group must match the number of axes used by the KINEMATICS. The axes must also be ascending order though they do not have to be contiguous.

Syntax

KINEMATIC_GROUP(index[, function])

 

KINEMATIC_GROUP(index, table offset, kinematic number, axis0 - axisn)

Parameters:

Index: INTEGER

Group index. Up to 8 robots can be set in the same controller. Valid index: 0..7

(optional) function: INTEGER

-1 = disable the kinematic group specified by index. All the axes in the group will be released.

table offset: INTEGER

Location in TABLE where the robot parameters are stored.

kinematic number: INTEGER

Robot type as defined by FRAME. See TrioBasic command FRAME for more information.

axis0 – axisn: INTEGER

Axes belonging to the robot in ascending order

Returned Value:

Group index [table_offset] [kinematic number]: axes TO={index [name] : TOOL_OFFSET parameters}  RF={index [name] : ROBOT_FRAME parameters}  OB={index [name] : OBJECT_FRAME parameters}  CO{active collision objects}

TO

Active Tool Offset (specified by index, name, and parameters that follow)

RF

Active Robot Frame (specified by index, name, and parameters that follow)

OB

Active Object Frame (specified by index, name, and parameters that follow)

CO

Total active collision objects in binary index wise. For example, if there are 4 collision objects (index 0, 1, 2, 3) and only collision object index 0 and 3 is active, CO will be of value 2 1 + 2 3 = 2 + 8, or {10}

The number of axes in KINEMATIC_GROUP must match the number of axes used by the kinematic number selected. The axes must be in ascending order but not necessarily contiguous.

When disabling a kinematic group, all axes of the group must be IDLE. If not, runtime error “Operation cannot be performed until IDLE” will be thrown.

Examples

Example 1

Check if RPS is set by displaying kinematic group index 0.

Terminal

>>kinematic_group(0)

0 [0] [19] : 10, 11, 12, 13, 14, 15 TO={0[TO_default]:0.00000, 0.00000, 0.00000, 0.00000, 0.00000, 0.00000} RF={0[RF_default]:0.00000, 0.00000, 0.00000, 0.00000, 0.00000, 0.00000} OF={0[OF_default]:0.00000, 0.00000, 0.00000, 0.00000, 0.00000, 0.00000} CO{3}

Example 2

Disable kinematic_group index 1.

Terminal

>> kinematic_group(0, -1)

Example 3

Remove collision object index 1.

Terminal

>>collision_object(-2, 1)

Example 4

Disable collision object index 0.

Terminal

>>collision_object(0, OFF)
