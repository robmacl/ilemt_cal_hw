Errors

RPS programs can generate errors due to errors in the program structure, or type of variables used, or as the result of a command being unsuccessfully executed.

Most errors are reported using a channel 9 status message which will be displayed as an error in Motion Perfect. Some commands return a value which indicates the error status.

In addition to error codes, supplementary information may be given in the axis status and the robot status, which are accessed using the AXISSTATUS and ROBOTSTATUS keywords.

All error codes are detailed in section TrioBASIC Errors , but RPS errors are redirected to this page to make it easier to locate RPS specific error codes.

Robot Log

A log system is provided to store in flash events that can happen at any time. Up to 2048 entries are stored as a cyclic buffer.

The stored data is provided along with the followings: entry index, date and time of the event, controller up time and message.

Syntax

ROBOT_LOG

ROBOT_LOG(function)

ROBOT_LOG(start_index[, end_index])

ROBOT_LOG(name)

Parameters:

-1

ROBOT_LOG(-1) deletes all entries

(optional) start_index

Starting log index to read

(optional) end_index

Ending log index to read

(optional) name

Name of destination file where to store robot log messages to.

>>robot_log
