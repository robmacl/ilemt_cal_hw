SLOT_NUMBER

Type:

Axis Parameter

Description:

Returns the SLOT number where the axis is located.  Axis numbers can be allocated to hardware in a flexible way, so the physical location of the axis cannot be found by the AXIS number alone.  SLOT_NUMBER returns the value from the BASE axis or if the AXIS (number) modifier is used, it returns the SLOT associated with that axis.

SLOT_NUMBER can be set for any virtual axis that requires time interpolated registration when the hardware allows it.

Example:

Example 1:

Use the command line terminal 0 to find the SLOT_NUMBER for an axis:

>>PRINT SLOT_NUMBER AXIS(12)

2

>> 

Example 2:

On startup, check the axis slot configuration of the MC464/MC664 is correct.

IF SLOT_NUMBER AXIS(0)<>-1 THEN

  PRINT “Warning – Built-in axis configuration incorrect”

  PRINT “Axis 0 expected for this application.”

ENDIF

Example 3:

Set up 2 virtual axes so that they may use time-interpolated registration inputs.  This requires the hardware features found in the MC4N, P876 and P871.

SLOT_NUMBER AXIS(10)=1

SLOT_NUMBER AXIS(11)=1

…

REGIST(21,4) AXIS(10) ' arm the registration using input 4

REGIST(21,5) AXIS(11) ' arm the registration using input 5
