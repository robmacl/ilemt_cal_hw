RUN

Type:

System Command

Syntax:

RUN ["program"[, process]]

Description:

Runs a named program on the controller. Programs can be RUN from another program.

A program can be run multiple times in different processes. You can use PROCNUMBER to help assign values in the program.

Programs will continue to execute until there are no more lines to execute, a HALT is typed in the command line, a STOP is issued or there is a run time error.

When a program is RUN there is a short delay while the program’s data area is initialised. Running a program is therefore always slower than calling a subroutine by GOSUB or calling a FUNCTION .

This delay is more pronounced when using GLOBAL.

Whenever a program is RUN, the GLOBAL and CONSTANT tables are searched for the names of already defined variables. If many GLOBAL/CONSTANT names are pre-defined, this may cause a delay in starting the program.

From V2.0298 onwards, when an IEC program is running, the multi-tasking is flat with no fast processes. Use SCHEDULE_TYPE.8 = 1 in MC_CONFIG to enable fast processes with IEC 61131-3.

Parameters:

program:

Name of program to be run. If not present the SELECT ed program is run

process:

Optional process number. (default highest available)

Examples:

Example 1:

From the MAIN program, run the STARTUP program on process 2 and wait for its completion:

RUN "STARTUP", 2

WA(50)  'wait for program to start

WAIT UNTIL PROC_STATUS PROC(2) = 0   'wait for program to complete

WDOG = ON

Example 2:

After STARTUP has completed the MAIN program will start other programs running in the highest available processes.

RUN "IO_CONTROL"

RUN "HMI"

RUN "SAUSAGE_CHOPPER"

Example 3:

SELECT the program STARTUP and run it on the command line.

>>SELECT "STARTUP"

STARTUP selected

>>RUN%[Process 21:Program STARTUP] - Running

>>%[Process 21:Line 238] (31) - Program is stopped

>>
