MOVE_PA_CONT

Type:

Axis Parameter

Description:

This parameter and MOVE_PB_CONT are two axis parameters which control what the system software does with MOVE_PA / MOVE_PB when the move becomes active or MOVE_PA_IDLE / MOVE_PB_IDLE when the axis becomes idle.

By changing the value of this parameter, the behaviour of the MOVE_PA/MOVE_PB and MOVE_PA_IDLE/MOVE_PB_IDLE will change.

Parameters:

 

mode:

1

Bits 0..11 of MOVE_PA set output #; Bit 12 sets ON/OFF. Output set to setting in MOVE_PA_IDLE when MTYPE = 0

2

MOVE_PA is copied into AXIS_DEBUG_A

3

Bits 0..11 of MOVE_PA set output #; Bit 12 sets ON/OFF.  Output set to setting in MOVE_PA_IDLE when VP_SPEED = 0 or MTYPE = 0

4

The bit pattern in MOVE_PA is copied to the outputs starting at a given index.  The bit pattern defined in MOVE_PA_IDLE is output when MTYPE = 0.

5

Analogue channel in MOVE_PA bits 32..39 set to 32-bit value in MOVE_PA bits 0..31 when move is started.

6

Analogue channel in MOVE_PA bits 32..39 set to 32-bit value in MOVE_PA bits 0..31 when move is started. MOVE_PA_IDLE is used when move is complete.

Mode 4 is available in firmware v2.0284 and later

Modes 5 and 6 are available in firmware v2.0339 and later

Examples:

Example 1:

Switching output number 10 during a move once the command is executed (using control parameter 1):

FOR a = 0 TO 1
    UNITS AXIS(a )= 10
NEXT a

SPEED = 10
MERGE = ON
LIMIT_BUFFERED = 16
ACC(SPEED * 20)
TRIGGER

 

MOVE_PA = 10

MOVE_PA_CONT = 1

MOVE_PA.12 = ON
MOVE(200)
MOVE_PA.12 = OFF
MOVE(20)
MOVE_PA.12 = ON
MOVE(100)
