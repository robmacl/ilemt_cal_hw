NW_NODE_INDEX

Type:

System Array (MC_CONFIG)

Syntax:

NW_NODE_INDEX(slot, alias_address) = value

Description:

This 2D array can be used to set the pointer to a block of VRs used by the EtherCAT node. It can be used to define a user specific Input Output map from different data sources including Boolean and Integer data within the EtherCAT node.

There is one VR mapped per PDO object, starting with the values from slave to master, ( eg slave actual values, DIN, status word, actual position etc.) then the values from master to slave ( eg slave target values, DOUT, control word, target position etc.)

Parameters:

slot:

The slot number of the EtherCAT port.  (0 on MC4N, MC6N and Flex-6 Nano)

alias_address:

The configured alias address of the slave device.

value:

0 to 65535

EtherCAT cyclic data is mapped to a block of VRs starting at this VR index. (MC664, MC6N, Flex-6 Nano)

0 to 4095

EtherCAT cyclic data is mapped to a block of VRs starting at this VR index. (MC4N)

Example:

A generic EtherCAT device is connected to the master and exchanges all its data via Vrs.  The base address of the VRs is 50 for outputs and 100 for inputs.  The user sets the actual addresses to be 1050 and 1100.  The device has configured alias address 4000.

NW_NODE_INDEX(0, 4000) = 1000
