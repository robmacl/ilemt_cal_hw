NODE_SYNC_OFFSET

Type:

System Array (MC_CONFIG)

Syntax:

NODE_SYNC_OFFSET(slot, node_address) = offset_value

Description:

This 2D array can be used to set the EtherCAT Sync0 offset time for any node on the network. This can be used to define a specific offset that applies to the defined node, leaving all other nodes at the default Sync0 offset value.  The default Sync0 offset may also be defined using ETHERCAT function $91.

The array is 2-dimensional, the first dimension is the master slot identifier, the second dimension is the position of the node within the master network.

Value:

Offset value in nanoseconds. This is a signed value and is the offset from the system value set by ETHERCAT function $91 or from the default system value of -70000 ns (nanoseconds).

ETHERCAT function $91 is scaled in microseconds ( μ s).

The default Sync0 offset is -70000 ns in V2.0287 and below. It is 0 ns in V2.0288 and higher.

Examples:

Example 1:

A Linmot C1250 drive requires a specific Sync0 offset of 200 μ s. There are other drives on the same EtherCAT network that do not require this offset. The Linmot drives are at nodes 2, 3, 4 and 5.  The default sync0 start value of 1000000 μ s is used, so the system Sync0 offset is 0 ns.

The default Sync0 offset can be checked with ETHERCAT($92) in MPv4 terminal #0

In MC_CONFIG

NODE_SYNC_OFFSET(0, 2) = 200000

NODE_SYNC_OFFSET(0, 3) = 200000

NODE_SYNC_OFFSET(0, 4) = 200000

NODE_SYNC_OFFSET(0, 5) = 200000

Example 2:

A network contains 3 x Kollmorgen AKD drives and one ABB Microflex e150 at node 3. The ABB drive requires the Sync0 offset to be -70 μ s. The Kollmorgen drives do not need this offset. The default sync0 start value is used, so the system Sync0 offset is 0 ns.

The default Sync0 offset can be checked with ETHERCAT($92) in MPv4 terminal #0

In MC_CONFIG

NODE_SYNC_OFFSET(0, 0) = 0

NODE_SYNC_OFFSET(0, 1) = 0

NODE_SYNC_OFFSET(0, 2) = 0

NODE_SYNC_OFFSET(0, 3) = -70000
