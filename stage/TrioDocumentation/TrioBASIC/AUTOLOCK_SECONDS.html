AUTOLOCK_SECONDS

Type:

System Command (MC_CONFIG only)

Syntax:

AUTOLOCK_SECONDS = value

Description:

The AUTOLOCK_SECONDS command is designed to automatically lock the controller following a disconnection with Motion Perfect, either when a certain amount of time elapsed, or immediately after the ethernet connection to Motion Perfect is disrupted.

This command can be activated within MC_CONFIG to enable the feature immediately on start-up, or later, either in a program or via the command line.

When a Motion Coordinator is locked, it is not possible to view, edit or save any programs and command line instructions are limited to those required to execute the program.

This feature is initiated once it is locked via Motion Perfect.

You should use Motion Perfect to LOCK and UNLOCK your controller.

Once locked, controller will remain locked following a power cycle.

To unlock the controller, the UNLOCK command should be entered using the same lock code number which was used originally to LOCK it.

The lock code number may be any integer and is held in encoded form. Once LOCKed, the only way to gain full access to the Motion Coordinator is to UNLOCK it with the correct code.  For best security the lock number should be 7 digits.

 Limitation on Flex6-Nano for immediate lock mechanism on ethernet disruption. This feature is currently not implemented on Flex6-Nano.

Parameters:

value:

Duration in seconds when controller is locked after being disconnected from Motion Perfect.

-1 = Default value. AUTOLOCK_SECONDS is disabled (as with any negative values)
