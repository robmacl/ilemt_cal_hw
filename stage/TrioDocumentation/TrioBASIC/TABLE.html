TABLE

Type:

System Command

Syntax:

value = TABLE(address)

array = TABLE(start address, last address)

TABLE(address[, data0..data35])

Description:

The TABLE command can be used to load and read back the internal TABLE values. As the table can be written to and read from, it may be used to hold information as an alternative to variables. A single global floating-point value can be read from an address or alternatively a range of floating-point values can be read from a range of addresses as an array (1D) of floating-point values that could be copied to another area of VR/TABLE memory or a locally defined array.

The table values are floating-point and can therefore be fractional.

You can clear the TABLE using NEW “TABLE”

Some TABLE elements are automatically backed up depending on the controller type. Use the FLASHTABLE command to force storing TABLE data.

Parameters:

value:

returns the value stored at the address

address:

The address of the first point of a write, or the single address to read

array

1D array of values read from a consecutive range of TABLE addresses

first address, last address

Range of TABLE addresses to read from as a 1D array

data0:

The data written to the address

data1:

The data written to address + 1

data2:

The data written to address + 2

…

 

data35

The data written to address + 35

This loads the TABLE with the following values, starting at address 100:

Example 2:

Use the command line to read the value stored in index 1000.

>>PRINT TABLE(1000)

1234.0000

>> 

Example 3:

Copy a range of TABLE locations to a local integer 2D array

DIM myints AS INTEGER(5, 20)

myints(2) = TABLE(1000, 1019) 'Row 2 will be populated

PRINT myints

Example 4:

Copy a range of TABLE locations to another area of VR and TABLE memory

TABLE(1000, TABLE(2000, 2199)) 'Copy points 2000..2199 to 1000

VR(0) = TABLE(100, 199) 'Copy points 100..199 to VR index 0
