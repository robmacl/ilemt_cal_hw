DRIVE_FE

Type:

Axis Parameter

Description:

Returns the value of following error calculated by a remote drive in position mode.  For this value to be active, the cyclic data transfer from the drive must be first configured to return the drive’s Position Error Actual value. For a drive connected by CANopen over EtherCAT (CoE) the value will be configured as part of the Process Data Object (PDO).

DRIVE_FE is a more precise value of the following error than FE. With a remote drive in position mode, the bus cycle time results in the Motion Coordinator FE value increasing with speed regardless of the true following error.

When DRIVE_FE is active, the DRIVE_FE_LIMIT must be set in the axis start-up as well as the normal FE_LIMIT.

When ABS(DRIVE_FE) > DRIVE_FE_LIMIT, the E flag, bit 8 of AXISSTATUS is set and a MOTION_ERROR will occur.

Value:

The drive position error returned in drive units.

Example:

Example1

Display the drive’s position error to Motion Perfect terminal 5.

PRINT #5,"Drive Position Error = "; DRIVE_FE AXIS(3)

Example2

Set up the axis to use the reported DRIVE_FE (Following Error Actual value) from the servo drive connected by EtherCAT.

MC_CONFIG:

DRIVE_PROFILE AXIS(2) = 1 'Select built-in profile with following error in the PDO

DRIVE_MODE = 1 'Select position mode (CSP)

STARTUP:

BASE(2)

UNITS = 2 ^ 21 / 5 '21 bit encoder. Ball screw with 5 mm pitch.
FE_LIMIT = 2.5 'Following error limit 2.5 mm

DRIVE_FE_LIMIT = 1.0 'Drive fe limit 1.0 mm

 

'Also set the internal position error limit in the drive

CO_WRITE_AXIS(2, $6065, 0, 7, -1, 2 ^ 20) 'Set to 0.5 x turn
