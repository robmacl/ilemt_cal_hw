ROBOT

Type:

Robotics Command

Syntax:

ROBOT(identity | function)

Description:

RPS can configure up to 8 robots in the same system. To let the program know what Robot it should direct its motion commands and other parameters to, a robot must be selected first by invoking ROBOT(identity ) command.

Once a Robot is selected, it will stay with that process until the movements produced by that process are finished and program stops. If a program stops (process finishes) then the Robot will not be released until buffers are idle. It is possible to check what process hold a particular Robot by reading ROBOT(identity) command.

Parameters:

identity:           

Robot unique identity number

Function

-1 = release robot from process. Note: releasing a robot must be in the same process as Robot(identity)

Examples:

Example 1:

Move Robot(0) between 2 defined points whilst input 303 is active. Release robot if input 303 is not active.

ROBOT ( 0 )

WHILE IN ( 303 ) = ON
     MOVEJ GTA ( "pInfeed_pt1" ) S:= 10 O:= "infeed"
     MOVEJ GTA ( "pOutfeed_pt1" ) S:= 10 O:= "outfeed"
WEND

ROBOT (- 1 )
