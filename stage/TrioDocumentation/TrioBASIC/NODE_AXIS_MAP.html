NODE_AXIS_MAP

Type:

System Array (MC_CONFIG)

Syntax:

NODE_AXIS_MAP(slot, node, axis_index) = value

Description:

This 3D array can be used to over-ride the drive to controller axis mapping for multi-axis drives of any EtherCAT sub-device. This can be used to define a user specific axis map to fix axes from different sources in place.

The array is 3-dimensional, the first dimension is the master slot identifier, the second dimension is the position of the node within that master network, the third dimension is the axis index in the drive starting from 0.

An error is raised if the axis requested is already in use when the EtherCAT protocol is started.

Value:

0

EtherCAT axis is allocated automatically (default)

>= 1

EtherCAT drive is located at this axis

Example 1:

Configure single axis drive TRIO DX4 from axis number 1 to axis number 20 and P374 2 servo axis from axis number 10 and 11 to axis number 10 and 11 respectively.
