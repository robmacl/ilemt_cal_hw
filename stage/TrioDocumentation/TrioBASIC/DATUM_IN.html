DATUM_IN

Type:

Axis Parameter

Alternate Format:

DAT_IN

Description:

This parameter holds a digital input channel to be used as a datum input.

The input used for DATUM_IN is active low.

Value:

-1

disable the input as DATUM_IN (default)

0-IO_Max

Input to use as datum input

Any type of input can be used, built in, Trio CAN I/O, CANopen, EtherCAT or virtual.

Example:

Example 1:

Set input 28 as the DATUM input for axis 0 then perform a homing routine with normally closed (NC) sensor.

DATUM_IN AXIS(3)=28

DATUM(4) AXIS(3) ' reverse until the datum switch is reached

WAIT IDLE AXIS(3)

Example 2:

Set up inputs for X and Y axes for homing.  The inputs are normally open (NO) proximity sensors therefore the inputs must be inverted.

BASE(x_axis)

INVERT_IN(205, 1)
DATUM_IN = 205


BASE(y_axis)
INVERT_IN(206, 1)
DATUM_IN = 206
