ROBOT_SP_MODE

Type:

Robotics Parameter

Syntax:

ROBOT_SP_MODE = value

Description:

The parameter puts the interpolation into a mode suitable for XYZ Cartesian robots with additional axes (UVW) that may control a tool head or “wrist”.

When ROBOT_SP_MODE is ON, the system calculates the path lengths for the XYZ group (e.g. axes 0,1,2) and the UVW group (e.g. axes 3,4,5) separately. It will time the move based on which one will take longer assuming the XYZ group takes speed from axis 0 and the UVW wrist group takes speed from axis 3. It does this by adjusting the "path length" so the U axis speed only has an effect during move calculation time.

INTERP_FACTOR should be set to 1.0 on all the axes in this mode.

This parameter is designed to work with very specific axis layouts. It may not have the desired effect if used more generally. Contact Trio if in doubt.

Value:

0

Interpolation uses all axes in the group to calculate the path.

1

Interpolation is in Robot mode 1

Examples:

Example 1:

So that motion may be performed on the wrist axes while keeping the XYZ position stationary, set ROBOT_SP_MODE = ON and run 6 axis moves as required in the sequence.

ROBOT_SP_MODE = 1

FOR ax = 0 TO 5

    INTERP_FACTOR AXIS(ax) = ON

NEXT ax

 

MOVEABS(0, 0, 0, 0, 0, 0)

MOVEABS(5, 15, 20, u1, v1, w1) 'All 6 axes are moved

MOVEABS(10, 20, 30, u1, v1, w1) 'Only the XYZ axes are moved

MOVEABS(10, 20, 30, u2, v2, w2) 'Only the wrist axes are moved

 

Setting INTERP_FACTOR to OFF on axes 3, 4 and 5 would result in the last move failing to run as expected.

Example 2:

Demonstration of how the XYZ and UVW can have their own speed setting.  The axes taking the longest time to complete their move will have control of the completion time for both sets of axes, i.e. the “Faster” axes will be made to run slower.

 

ROBOT_SP_MODE = 1

 

FOR ax = 0 TO 5

   INTERP_FACTOR AXIS(ax) = ON

NEXT ax

 

MOVE_COUNT = 0

 

UNITS AXIS(0) = 1000 'Gives mm on XYZ

UNITS AXIS(1) = 1000 'Gives mm on XYZ

UNITS AXIS(2) = 1000 'Gives mm on XYZ

 

UNITS AXIS(3) = 10000 / 360 'Gives deg on wrist

UNITS AXIS(4) = 10000 / 360 'Gives deg on wrist

UNITS AXIS(5) = 10000 / 360 'Gives deg on wrist

 

SPEED AXIS(0) = 100 'mm / sec

SPEED AXIS(3) = 60 'deg / sec

ACCEL AXIS(0) = 5000

DECEL AXIS(0) = 5000

 

TRIGGER

WA(20)

 

DEFPOS(0, 0, 0, 0, 0, 0)
