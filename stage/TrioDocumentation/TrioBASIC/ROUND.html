ROUND

Type:

Mathematical Function

Syntax:

value = ROUND(expression)

value = ROUND(expression, precision)

Description:

The ROUND function returns the floating-point value rounded (towards zero) to the specified precision with the result being floating-point.

If the precision is omitted or is zero, the value is rounded to the nearest whole value.

Despite the result being floating-point, integer values up to ±(2 53-1) are represented accurately, but bear in mind that values outside of this range will round less accurately as they are subject to the same inaccuracies as using INT with an offset of ±0.5

If the precision is positive, the fractional part is rounded to the specified precision (number of decimal places, i.e. to the nearest tenth, hundredth, thousandth, etc.).

If the precision is negative, the integer part value is rounded to the specified precision (to the nearest ten, hundred, thousand, etc.).

When rounding with a non-zero precision, the result is rounded using INT(<value> * 10 ^ <precision> + 0.5) / 10 ^ <precision> for positive values and INT(<value> * 10 ^ <precision> - 0.5) / 10 ^ <precision> for negative values.

When the precision is either omitted or is zero, this is equivalent to INT(<value> + 0.5) for positive values, and INT(<value> - 0.5) for negative values.

Parameters:

expression:

The floating-point value that requires rounding

precision:

The precision with which to round to (zero assumed if omitted)

Examples:

Example 1:

Print a floating-point value rounded to the nearest whole value on the command line

>>PRINT ROUND(PI)

3.0000

Example 2:

Print a floating-point value with the integer part rounded to a varying number of decimal places on the command line

>>PRINT ROUND(PI, 1), ROUND(PI, 2), ROUND(PI, 3), ROUND(PI, 4)

3.1000     3.1400     3.1420     3.1416

Example 3:

Print a floating-point value with the integer part rounded to a varying number of places on the command line

>>PRINT ROUND(1000 * PI, -1)

3140.0000

>>PRINT ROUND(1000 * PI, -2)

3100.0000

>>PRINT ROUND(1000 * PI, -3)

3000.0000

>>PRINT ROUND(1000 * PI, -4)

0.0000
