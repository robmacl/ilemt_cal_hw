CONSTANT

Type:

System Command

Syntax:

CONSTANT ["name"[, value]]

Description:

CONSTANTS can be declared in the controller which are then available to all programs. They must be declared on startup and the program declaring CONSTANTs must also be the ONLY process running at power-up.

Once a CONSTANT has been assigned it cannot be changed, even if you change the program that assigns it.

Model number

Max number of CONSTANTs

MC464

1024

MC403(Z), MC405, Euro404, Euro408, MC4N-ECAT, MC4N-RTEX

4096

MC664(X), MC508, Flex-6 Nano, Flex-6X Nano, MC6N-ECAT

8192

While developing you may wish to clear or change a CONSTANT. You can clear a single CONSTANT by using the first parameter alone. All CONSTANTs can be cleared by issuing CONSTANT. You can view all CONSTANTS using LIST_GLOBAL .

Whenever a program is RUN, the GLOBAL and CONSTANT tables are searched for the names of already defined variables.  If a large number of GLOBAL/CONSTANT names are pre-defined, this may take several seconds.  When using the GLOBAL and CONSTANT system is it not advised to depend on temporarily running and then stopping programs.  It is best to run all the required programs when the machine starts and leave them in that running state.

Parameters:

name:

Any user-defined name containing lower case alpha, numerical or underscore (_) characters.

value:

The value assigned to the name.

Examples:

Example 1:

Declare 3 CONSTANTs and use them within the program

CONSTANT "nak",$15

CONSTANT "start_button",5

CONSTANT "led1",8

 

IF IN(start_button)=ON THEN OP(led1,ON)

IF key_char = nak THEN GOSUB no_ack_received

Example 2:

Use the command line to clear a defined constant

>>CONSTANT "NAK"

>> 

Example 3:

Use the command line to clear all defined constants

>>CONSTANT

>>
